---
title: "Agent Grid: Priority-Sorted Session Cards per Project"
description: "The agent grid displays every session for the active project as a priority-sorted card, surfacing the sessions that need your attention first."
---

The agent grid is the main workspace in CC-GUI. It occupies the center of the layout and shows every session belonging to the currently selected project as an interactive card. Cards are sorted automatically by priority so the sessions that need your attention always appear at the top — you never have to hunt for what's urgent.

## How the grid is sorted

The orchestrator server analyses each session's terminal output continuously and assigns a numeric priority from 0 to 4. The grid re-sorts in real time whenever priorities change. Within the same priority tier, sessions are further ranked by how long they have been waiting and, as a tiebreaker, alphabetically by name.

### Priority levels

| Level | Name | Value | Trigger | Action hint |
|-------|------|-------|---------|-------------|
| Highest | **CRITICAL** | 4 | Permission prompts, `(Y/n)` confirmations | Respond now |
| | **HIGH** | 3 | Waiting for user input (`>` / `❯` prompt), errors, non-zero exit | Needs attention |
| | **MEDIUM** | 2 | Task completed — output contains `Done`, `✓`, `Successfully`, etc. | Review when ready |
| | **LOW** | 1 | Actively working — thinking, running a tool, writing, building | No action needed |
| Lowest | **IDLE** | 0 | No detectable activity | — |

<Note>
  The grid auto-sorts on every priority update. CRITICAL sessions always rise to the top of the list regardless of when they were created or how long other sessions have been waiting.
</Note>

## What each card shows

Each card is a snapshot of one session. Reading from top to bottom:

- **Priority dot** — a colour-coded dot that blinks when priority is HIGH or CRITICAL.
- **Pokémon name and session name** — the starter name (e.g., Bulbasaur) plus the human-readable label you gave the session.
- **Protected badge** — `PROT` appears for sessions running a protected agent command (e.g., `codex`).
- **Granular state** — a fine-grained live label such as `thinking`, `running tool`, `retry 2/3`, `done`, or `exited`.
- **Reason** — a short phrase from the orchestrator explaining the current priority (e.g., *"Confirmation needed"*, *"Rate limited"*, *"Task completed"*).
- **AI summary** — a one-line summary generated by the AI orchestrator describing what the session is working on.
- **Git info** — commit count and changed-file count for the session's branch, plus `overlap` or `CONFLICT w/ <name>` warnings when two sessions touch the same files.
- **PR controls** — a **CREATE PR** button when the session has unpushed commits, a live status badge while a PR is being created, a **PR ✓** link once the PR exists, or **PR FAIL** if creation errored.

## Opening a session

Click any card to open the full-screen terminal overlay for that session. The PTY is already running — you are dropped straight into the live output.

<Tip>
  You can also press **S** to quick-spawn a new session on the active project without opening a dialog. The new session opens immediately in the terminal overlay.
</Tip>
