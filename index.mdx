CC-GUI gives you a single browser window to spawn, monitor, and interact with multiple Claude Code sessions across all your projects. Every session runs in a real PTY on your machine and streams live output into an xterm.js terminal — no polling, no abstractions.

## What CC-GUI does

CC-GUI connects your local file system to a retro-themed dashboard where each Claude Code session appears as a card in an **agent grid**. The grid sorts sessions automatically by priority — sessions waiting for your input or hitting errors float to the top so you never miss a prompt.

## Get started in three steps

<Steps>
  <Step title="Clone and install">
    ```bash
    git clone https://github.com/Haroutkhac/CC-GUI.git
    cd CC-GUI
    npm install
    cp data/store.example.json data/store.json
    ```
  </Step>
  <Step title="Start the dev server">
    ```bash
    npm run dev
    ```
    This starts the Express API on port 3456 and the Vite frontend on port 5173. Open **http://localhost:5173** in your browser.
  </Step>
  <Step title="Add a project and spawn a session">
    Click **+ TABLE** (or press `N`) to add a project directory, then press `S` to spawn a Claude Code session. Click the session card to open the terminal.
  </Step>
</Steps>

<Note>
  CC-GUI runs in safe mode by default, binding to `127.0.0.1` only. It is designed for local use and should not be exposed to the public internet.
</Note>
